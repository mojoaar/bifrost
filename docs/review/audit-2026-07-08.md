## Security Findings Detail

### CRITICAL

- **C1** PUT `/api/v1/settings` has NO auth — anyone can write arbitrary settings including `git.token`, `ai.key.*`
- **C2** POST\|PUT\|DELETE `/api/v1/users/*` has NO auth — attacker can create admin accounts
- **C3** POST\|PUT\|DELETE `/api/v1/tags/*` has NO auth
- **C4** POST `/api/v1/git/pull` and `/api/v1/git/push` have NO auth
- **C5** Hardcoded JWT dev secrets in `lib/auth/token.ts:13-20` — if env vars not set, attacker can forge tokens
- **C6** Middleware `WRITE_API_PATTERNS` only covers posts/pages/media/ai — 8 other route families bypass it
- **C7** MCP `get_settings` tool + `bifrost://settings` resource return UNREDACTED secrets
- **C8** MCP HTTP server (`/sse`) has NO auth — anyone can call tools incl. `delete_post`, `ai_assist`
- **C9** GET `/api/v1/posts` and GET `/api/v1/pages` return drafts without auth

### HIGH

- **H1** `requireUser` trusts `x-user-id`/`x-user-role` headers without verification
- **H2** AI chat route has ZERO auth — middleware passes `bfk_` through unverified, handler checks only `ai.enabled`
- **H3** No password complexity enforcement on setup/user creation (only profile checks 8 chars)
- **H4** No rate limiting on any endpoint
- **H5** 9+ routes leak raw error details via `String(err)`
- **H6** Logout clears wrong cookie name (`bifrost_refresh_token` vs `bifrost_refresh`)
- **H7** `bcrypt` native binding (not `bcryptjs`) — potential edge-runtime crash

### MEDIUM

- **M1** API key `toBase62` uses `b%62` (slight bias)
- **M2** No security headers (CSP, HSTS, X-Content-Type-Options)
- **M3** No CSRF protection
- **M4** Setup TOCTOU race

### LOW

- **L1** `__SET__` placeholder recognizable/predictable
- **L2** OpenAPI spec exposed without auth
- **L3** Analytics endpoint vulnerable to spam (no rate limit)
