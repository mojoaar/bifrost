# v1.18.0 — Security Hardening, Admin UX & Theme Editing

## Scope

## Part 0: Bugfix — Featured Image Gap on List Cards

**File:** `themes/bifrost-terminal/list.tsx:94`

**Bug:** `w-[calc(100%+2.5rem)]` — CSS `calc()` requires spaces around operators. Without them, the declaration is invalid and the browser ignores it, falling back to `auto` width (the image's natural width). This creates a visible gap on the right side of the card.

**Fix:** Use Tailwind arbitrary value underscores (converted to spaces): `w-[calc(100%_+_2.5rem_+_2px)]`. Also update negative margins to account for the card's 1px border, so the image sits flush edge-to-edge.

```
className="-mx-[calc(1.25rem_+_1px)] -mt-[calc(1.25rem_+_1px)] mb-4 h-40 w-[calc(100%_+_2.5rem_+_2px)] object-cover rounded-t-md"
```

## Part 1: Security Fixes

### M1 — API Key Modulo Bias
**File:** `lib/auth/api-key.ts:29-33`

`toBase62()` maps each random byte via `b % 62`. Since 256 is not evenly divisible by 62, the first 8 characters get ~2x probability. Switch to rejection sampling: skip bytes >= 248, map remainder via `b % 62`.

### H7 — bcrypt Native Binding
**Files:** `package.json`, `lib/auth/password.ts`, `lib/auth/api-key.ts`

Swap `bcrypt` (native C++ binding) for `bcryptjs` (pure JS, identical API). Remove `@types/bcrypt` devDependency. No runtime behavior change.

### M3 — CSRF Protection
**New file:** `lib/auth/csrf.ts`

`validateCsrf(request: NextRequest): boolean` — compare `Origin` or `Referer` header host against request host. Add to `app/api/v1/auth/refresh/route.ts` — reject with 403 if validation fails. No client changes needed.

### M4 — Setup TOCTOU
**File:** `app/api/v1/setup/route.ts`

Wrap the `SELECT` check + `INSERT` in `db.transaction()` to serialize concurrent setup requests.

### L1 — `__SET__` Scoping
**File:** `app/api/v1/settings/route.ts:35-36`

PUT handler currently skips `__SET__` for all keys. Change to only skip for `isSecretKey(key)`.

### L2 — OpenAPI Docs Gate
**Files:** `app/api/v1/openapi.json/route.ts`, `app/api/docs/route.ts`

Gate both GET endpoints behind env var `BIFROST_API_DOCS_ENABLED`. Default `true` in dev, should be `false` in production. Return 404 when disabled.

## Part 4: Theme Editing

### 4A — Custom CSS Editor

**New page:** `app/admin/themes/custom-css/page.tsx`
- CodeMirror CSS editor (no markdown toolbar, just plain text with syntax highlighting)
- Save button (Cmd/Ctrl+S)

**New API:** `app/api/v1/themes/custom-css/route.ts`
- `GET` — read `themes/bifrost-terminal/custom.css` (return empty string if file doesn't exist)
- `PUT` — write to `themes/bifrost-terminal/custom.css`

**Theme loading:** Add `<link rel="stylesheet" href="/api/v1/themes/custom-css" />` to theme layout (or inline via `<style>` in layout component). The custom CSS is injected at runtime — no rebuild needed.

### 4B — Theme Settings Toggles

**Extend:** `app/admin/themes/page.tsx`

Settings keys:
- `theme.list.featuredImage` (boolean, default `true`) — show/hide featured image hero strip on list cards
- `theme.list.readingTime` (boolean, default `true`) — show/hide reading time on list cards
- `theme.post.authorBio` (boolean, default `true`) — show/hide author bio footer on posts

**Theme props:** Pass settings to `list.tsx` and `post.tsx` via Props. Components gate the relevant sections behind these toggles.

**UI:** `<Toggle>` component for each setting on the themes admin page.

### 4C — Theme File Editor

**New page:** `app/admin/themes/files/page.tsx`
- File tree sidebar (directory listing of `themes/bifrost-terminal/`)
- CodeMirror editor for selected file (mode detected from extension: `.tsx`/`.css`/`.json`)
- Save button (Cmd/Ctrl+S)
- Warning banner: "Dev server restart required to apply template changes"

**New API:** `app/api/v1/themes/[name]/files/route.ts`
- `GET ?path=<relative>` — read file content
- `PUT` — write file content (body: `{ path, content }`)
- Reject paths outside theme directory (path traversal protection)
